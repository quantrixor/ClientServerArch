API Server Documentation

This document describes the endpoints available in our API server and how to interact with them.

GET /api

Returns API documentation.

Response:

HTTP/1.1 200 OK
Content-Type: text/html

[HTML Content of API Documentation]

GET /api/MedicalCards

Returns a list of all medical cards.

Response:

HTTP/1.1 200 OK
Content-Type: application/json

[
    {
        "Id": 1,
        "Code": "123456",
        "PathQRCode": "path/to/qr/code1.png"
    },
    {
        "Id": 2,
        "Code": "654321",
        "PathQRCode": "path/to/qr/code2.png"
    }
]

POST /api/MedicalCards

Adds a new medical card. Requires a unique code and the path to its QR code.

Request:

{
    "Code": "unique_code",
    "PathQRCode": "path/to/qr/code.png"
}

Response:

HTTP/1.1 201 Created
Content-Type: application/json

{
    "Message": "New medical card added successfully."
}
HTTP/1.1 409 Conflict
Content-Type: application/json

{
    "Message": "The code already exists."
}
